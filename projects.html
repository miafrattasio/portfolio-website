Juice Shop Penetration Testing

This project focused on a penetration test analysis of the OWASP Juice Shop, a deliberately insecure web application used for training and demos on the OWASP Top 10 vulnerabilities. The analysis, that I presented to the Brown Brothers Harriman Cyber Working Group, demonstrated real-world exploits using the Burp Suite tool. Key vulnerabilities I exploited included Broken Access Control, Injection, and Vulnerable and Outdated Components. My favorite exploit I completed was a NoSQL Manipulation attack to update multiple product reviews to "Hacked by Mia" simultaneously.

Technologies and Skills: Burp Suite, Broken Access Control, Injection, Vulnerable and Outdated Components, CSRF, IDOR, SQL Inejction, NoSQL Manipulation

UI State Replication: A Zero-Downtime Patching Solution

This project addresses the downsides of software patching downtime. The approach, designed to run locally on endpoint devices to avoiding high costs of LLMs, captures user actions and replays them on a newly patched VM in real-time. The program also validates progress and realigns when non-deterministic elements are introduced. This work provides a scalable, privacy-preserving, and accessible solution for enterprise endpoint security updates.

Street Heat: Urban Heat Monitoring in Andalusia

Urban Heat Islands (UHIs) intensify climate-related challenges in densely populated regions like Andalusia, Spain. This study addresses previous data collection gaps by developing a low-cost, Arduino-based portable device that measures environmental variables such as temperature, humidity, and light intensity. Book Collection

This project is a full-stack web application designed to allow individual users to manage and track their personal book collections. The core functionality centers on secure, user-specific data management, starting with a combined User Authentication process that handles both registration and login, with username and password pairs stored in a dedicated users collection in MongoDB. Once authenticated, users can perform all standard CRUD (Create, Read, Update, Delete) operations on their book entries. These books are stored in a separate books collection and are explicitly linked to the user via their username, ensuring personalized collections. Additionally, the application provides automatic Reading Status Tracking, dynamically setting a book's status to "Read" if a rating (1-5) is provided by the user, or "Not Read" if the rating field is left empty.

Technologies and Skills: Node.js, Express.js, MongoDB, HTML, Javascript, Tailwind CS
